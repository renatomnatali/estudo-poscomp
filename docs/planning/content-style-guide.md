# Guia Editorial de Conteúdo Didático (POSCOMP)

## Objetivo
Padronizar produção de tópicos para garantir clareza, objetividade e aderência à prova.

## Estrutura obrigatória por tópico
1. Camada essencial (explicação curta para prova).
2. Camada avançada (formalismo e nuances).
3. Dois exemplos resolvidos passo a passo.
4. Uma aplicação real de computação.
5. Quick-check com feedback explicativo.
6. Referências externas confiáveis.

## Regras de escrita
1. Português brasileiro com acentuação correta.
2. Frases curtas e termos técnicos precisos.
3. Evitar redundância (“resposta incorreta + gabarito + ...”).
4. Explicação sempre orientada a decisão de alternativa.

## Padrão de exemplos
1. Problema.
2. Estratégia.
3. Solução comentada.
4. Takeaway objetivo.

## Padrão de aplicação
1. Contexto de software/computação.
2. Como o conceito é usado na prática.
3. Benefício ou limitação na engenharia.
